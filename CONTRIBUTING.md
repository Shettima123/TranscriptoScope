# Contributing

Contributions are welcome.

Good first contributions include:

- installation feedback;
- documentation fixes;
- small reproducible bug reports;
- additional public example datasets;
- validation tests;
- organism resource improvements; and
- usability improvements for non-programming users.

Before proposing a code change, please open an issue describing the intended change and the dataset or workflow it affects. Changes that alter statistical behavior should include a small validation example and explain whether existing outputs are expected to change.

## Development Notes

TranscriptoScope is an R/Shiny application. Core analysis helpers live in `R/deseq_helpers.R`, while interface logic lives in `app.R`.

## Code Of Conduct

Be respectful and practical. The project is intended to help researchers complete transcriptomic analysis with less installation and scripting burden.
