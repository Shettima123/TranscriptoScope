# Security Policy

## Supported Versions

The latest public release is the supported version for bug fixes and security review.

## Reporting A Security Issue

Please do not open a public GitHub issue for a security problem. Contact the repository owner privately through the contact channel listed on the project page.

Useful details include:

- affected version;
- operating system;
- a description of the issue;
- steps to reproduce; and
- whether the issue involves local file access, package installation, or external network access.

TranscriptoScope is a local research application and is not intended to process protected or regulated clinical data without local institutional review.
